# 历史项目 Retrofit Playbook

## 目标

在不打断现网交付的前提下，将历史项目逐步升级到规范驱动的 AI 协作模式。

## 五步迁移

1. 资产盘点：模块、接口、测试、质量债务、线上风险。
2. 差距评估：对照本仓库模板识别差距。
3. 分批接入：先文档与规则，再门禁与结构测试。
4. 灰度验证：在低风险模块先 enforce。
5. 全量启用：形成统一门禁与流程。

## 推进策略

- 第一期：`observe`（2~4 周）
- 第二期：`warn`（2~6 周）
- 第三期：`enforce`（持续）

## 成功标准

- 关键模块完成结构测试约束。
- API 文档同步率 100%。
- 功能资产文档有持续更新。
- 线上问题率与回归缺陷率下降。
