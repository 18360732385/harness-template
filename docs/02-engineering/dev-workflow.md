# 工程研发流程（标准版）

## 流程

1. 需求澄清：确认边界、验收标准、风险。
2. 设计输出：完成技术方案、数据模型、接口契约。
3. 执行计划：拆分任务并定义验证方式。
4. 编码实现：按 plan_vs_impl 严格对齐实现。
5. 测试验证：单元、集成、回归。
6. 文档同步：API 文档与 `harness-collab/func.md`。
7. 复盘沉淀：问题归因与规范更新。

## 分支建议

- `feature/*`：功能开发
- `fix/*`：缺陷修复
- `refactor/*`：重构治理

## 提交要求

- 提交信息要说明“为什么改”。
- 每次提交保持单一意图。
- 不提交敏感配置与凭证信息。

## 提交前检查列表

- [ ] `mvn clean verify` 通过。
- [ ] `harness-collab/exec-plans` 中 plan_vs_impl 已更新。
- [ ] API 文档已同步（如有接口变更）。
- [ ] `harness-collab/func.md` 已同步（如有能力变更）。
- [ ] 风险与回滚内容已填写。

## PR 评审检查点

- 需求、设计、计划是否可追溯。
- 分层依赖是否符合架构约束。
- 测试证据是否覆盖关键路径。
- 文档同步与发布风险是否闭环。
