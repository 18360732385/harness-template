# 工程研发流程（标准版）

## 流程

1. 需求澄清：确认边界、验收标准、风险。
2. 设计输出：完成技术方案、数据模型、接口契约。
3. 执行计划：拆分任务并定义验证方式。
4. 编码实现：按 plan_vs_impl 严格对齐实现。
5. 测试验证：单元、集成、回归。
6. 文档同步：API 文档与 `func.md`。
7. 复盘沉淀：问题归因与规范更新。

## 分支建议

- `feature/*`：功能开发
- `fix/*`：缺陷修复
- `refactor/*`：重构治理

## 提交要求

- 提交信息要说明“为什么改”。
- 每次提交保持单一意图。
- 不提交敏感配置与凭证信息。
